General Definitions

 

OCCT general definitions for Data Exchange include several enumerations and classes. These are used by both the IGES and STEP data exchange interfaces.

To define translation parameters and file headers, you can use:

- Interface_InterfaceModel

- Interface_Static

To manage Message display, use class Mesage_Messenger.

To define the type of analysis of the source file, and to ensure the success of the loading operation, you use the following enumerations from the IFSelect package:

- PrintCount

- ReturnStatus

To read and write attributes such as names, colors, layers for IGES and STEP and validation properties and structure of assemblies for STEP, you can use an XDE document.

 

It is possible to learn more about XDE documents from XDE User's guide
